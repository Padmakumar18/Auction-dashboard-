EXAMPLES:
=========

1. /login (Public Standalone)
   User → RouteWrapper → Standalone + Public → No Layout, No Auth

2. /register (Public Standalone)
   User → RouteWrapper → Standalone + Public → No Layout, No Auth

3. /teams (Protected with Layout)
   User → RouteWrapper → Not Standalone, Protected → Auth Required + Layout

4. /player-registration (Conditional)
   Logged In User → RouteWrapper → Conditional → Show WITH Layout
   Not Logged In → RouteWrapper → Conditional → Show WITHOUT Layout

5. /player-registration-enhanced (Conditional)
   Logged In User → RouteWrapper → Conditional → Show WITH Layout
   Not Logged In → RouteWrapper → Conditional → Show WITHOUT Layout

USE CASES:
==========

1. Admin registering a player:
   - Logs in first
   - Navigates to /player-registration
   - Sees full dashboard with navigation
   - Can easily switch to other pages

2. Public user registering:
   - Goes directly to /player-registration
   - Sees only the form (no distractions)
   - Clean, focused experience
   - No login required

3. Sharing registration link:
   - Share /player-registration URL
   - Works for both logged-in admins and public users
   - Adapts automatically based on auth status
